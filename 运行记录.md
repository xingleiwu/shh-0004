## 技术实现原理

### 游戏状态机
```
未开始 → 进行中 ↔ 已暂停
   ↓        ↓
 退出     已胜利(结算弹窗)
```

### 卡牌数据结构
```javascript
{
  id: 0-63,           // 卡牌唯一ID
  value: '🐶',        // 卡牌正面emoji
  flipped: boolean,   // 是否已翻开
  matched: boolean    // 是否已配对
}
```

### 配对逻辑
1. 点击第一张卡 → 标记 flipped，加入 flipped 数组
2. 点击第二张卡 → 同上，moves+1，加锁防止连点
3. 对比 value:
   - 相同 → 标记 matched，加入 matched 数组，解锁
   - 不同 → 延时1秒后翻回去，双人模式切换玩家，解锁
4. 检查 `matched.length === cards.length` → 胜利

### 计时机制
- 每250ms更新一次显示，保证流畅性
- 暂停时不更新 elapsed，恢复时重新校准 startTime
